❓ ЧАСТО ЗАДАВАЕМЫЕ ВОПРОСЫ

Q: Какую модель я использую?
A: models/gemma-3-1b-it (Gemma 3, libre, быстрая)
   Это альтернатива gemini-2.0-flash (которые исчерпала квоту)

Q: Почему не gemini-2.0-flash?
A: Ваша бесплатная квота на эту модель исчерпана.
   Используется gemma-3 вместо неё - аналогичная по качеству.

Q: Как добавить Groq для голоса?
A: /setup в чате и отправить ваш Groq API key
   Или добавить в .env: GROQ_API_KEY=...

Q: Где сохраняется история?
A: В SQLite базе: /workspaces/Temka_bot/data/bot.db

Q: Как удалить сессию?
A: /sessions → выбрать сессию → нажать ❌

Q: Сколько сессий одновременно?
A: Неограниченно, но максимум 5 запросов в очереди.

🎓 ТЕХНИЧЕСКИЕ ДЕТАЛИ

Архитектура:
  aiogram v3 (async Telegram Bot Framework)
  + google-genai (Google Gemini SDK)
  + SQLite3 (история сессий)
  + asyncio (асинхронная обработка)

Поток данных:
  Telegram Message
    ↓
  main.py (handle_message)
    ↓
  gemini_runner.py (run_gemini)
    ↓
  Google Gemini API (REST)
    ↓
  db.py (сохранение в SQLite)
    ↓
  main.py (ответ в Telegram)

Управление сессиями:
  - Каждому пользователю: sessions (таблица БД)
  - Каждой сессии: history (сообщения user + model)
  - История передаётся в API для контекста

Очередь запросов:
  - Max 5 одновременно
  - Остальные ждут в очереди
  - FIFO (First In First Out)
